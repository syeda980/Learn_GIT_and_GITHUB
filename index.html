What is Git?

So far the problem is considered one should need a platform where all the changes can be tracked. Here comes the savior
Git is a DevOps used for source code management. It is one of the most prominent version control systems (VCS) today that is widely used to handle small and large projects efficiently.Version control systems are software tools that help software teams manage changes to source code over time. It helps in tracking changes in source code, enabling different people to collaborate on different parts of the same program.

Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.

What is GitHub?

GitHub is an online software development platform used for storing, tracking, and collaborating on software projects. It enables developers to upload their own code files and to collaborate with fellow developers on open-source projects. GitHub also serves as a social networking site in which developers can openly network, collaborate, and pitch their work.

Git Commands

Here are some git basic git commands.

  • git init
  • git config
  • git add
  • git status
  • git commit
  • git log
  • git clone
  • git push
  • git pull
  • Branches
  • git merge

We will get to know about these one by one.

  • git init

    git init is used to create new Repository. It can be used to create a empty repository or convert an existing, unversioned project to a Git repository. This is typically the first command you'll perform in a new project.

    This will create a folder named .git inside your project folder.It contains all of the necessary Git metadata for the new repository. This metadata includes subdirectories for objects, refs, and template files Since this is a hidden folder, Incase it is not visible, Go to view option from top menu in your explorere and check mark the hidden items option. After that the folder will be visible.


  • git config

    The git config command is a convenience function that is used to set Git configuration values on a global or local project level. It will modify the configuration file.Some common configuration settings like email, username, and editor can be set on local or global level of project.

    This example writes the value your_email@example.com to the configuration name user.email and Your Name to the confiuration name user.name. It uses the --global flag so this value is set for the current operating system user.


  • git add

    The git add command adds a change in the working directory to the staging area. It tells Git that you want to include updates to a particular file in the next commit.It is a way to record versions of a project into the repository's history. git add . add all the files in one go to staging area

    git add (fileName) adds specific file to staging area.

  • git status

    The working directory and staging area's status are shown by the git status command. It enables you to identify which files are not being tracked by Git as well as which changes have been staged and which have not. Here git status shows that there are changes in mentioned file which are not being tracked by git.

  • git commit

    Now after adding files, we are ready to move from stage to commit for our repository. Adding commits keep track of our progress and changes as we work. Git considers each commit change point or "save point". It is a point in the project you can go back to if you find a bug, or want to make a change. When we commit, we should always include a message. By adding clear messages to each commit, it is easy for yourself (and others) to see what has changed and when. To add message -m option is used.

  • git log

    git log shows the history of all the changes made in master branch or specified branch

  • git clone

    git clone is a Git command line utility which is used to target an existing repository and create a clone, or copy of the target repository. The original repository can be located on the local filesystem or on remote machine accessible supported protocols. The git clone command copies an existing Git repository. Here the shaded area contains the https link of your remote repository. you can copy the address from github website.

  • git push

    The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repository.After a local repository has been modified a push is executed to share the modifications with remote team members. Here git push add all the changes to remote repository.

  • git pull

    Using the git pull command, you may quickly update your local repository with the content you just downloaded from a remote repository. In Git-based collaboration work processes, merging remote upstream changes into your local repository is a typical task.

  • Branched

    What is branching?

    Branching is a feature available in most modern version control systems.Branching can also be done in git.Brancheds are useful when you are not sure on the changes you are going to make in the files is correct or not. Once it is confirmed that the changes are right you can link/merge that branch to the main branch that is the master branch.

    The illustration depicts a repository with two separate lines of development, one for a quick feature and the other for a more involved one. The ability to work on both of them simultaneously is made possible by developing them in separate branches, which also prevents shady code from entering the main branch.

    You can use the git branch command to create, list, rename, and delete branches. It does not allow you to switch between branches or reconstruct a forked history. As a result, git branch is tightly integrated with the git checkout and git merge commands.

    • git branch (branchName)
    Here branch1 is the name of the branch we have created:

    • git branch
    Executing git branch gives you the way to know where the HEAD is pointing to. * shows the current branch.

    • git checkout (branchName)
    Excuting git checkout lets you switch between branches. Here now HEAD points to branch2

    • git merge
    The git merge command combines the independent lines of development created by git branch into a single branch.
